Vue系列（十六）：自定义指令和过滤器

发表于 2018-05-27 | 分类于 前端 ， Vue |
字数统计 1,479 字 | 阅读时长 6 分钟

Vue 本身提供了很多例如v-if ，v-for等等的指令，用于对绑定的DOM元素进行各种各样的操作。在需要时，我们也可以自定义一些指令，从而实现一些操作逻辑的复用。

在使用双花括号或者v-bind指令绑定一些数据时，我们有时会需要对这些数据进行一些格式化操作，此时，我们可以自定义一个过滤器，方便快捷的完成这个需求。

本篇博客，就主要来介绍以下，Vue的自定义指令和自定义过滤器。

自定义指令

钩子函数

在上面的组件注册中，我们提到指令有一个options对象，用于配置指令。在options对象中，我们可以使用此对象提供的指令作用各阶段的钩子函数，来实现指令的功能。

指令主要有以下几个钩子函数：

  • bind

    在指令绑定到DOM元素时调用，只会被调用一次。可以在此处执行指令的初始化设置。

  • insert

    在指令绑定的DOM元素插入到父节点时调用。要注意的是，插入到父节点时父节点并不一定在文档中。

  • update

    指令所在的组件更新时调用。

  • componentUpdated

    指令所在的组件及其子组件全部更新后调用。

  • unbind

    指令解除绑定时调用，同bind类似，只会被调用一次。

以上是指令配置对象提供给我们的指令钩子函数。

这些钩子函数，都会被传入相同的参数，即el，binding，vnode 和 oldVnode。

各参数的具体含义如下：

  • el

    指令所绑定的DOM元素对象，可以用来直接操作绑定到的DOM。

  • vnode

    Vue编译生成的虚拟节点。作为抽象节点，VNode定义了很多属性，来模拟真实DOM节点。具体可以点击此处查看

  • oldVnode

    更新之前的vnode虚拟节点，在update和componentUpdated钩子中可用。

  • binding

    一个对象，包含了和指令有关的一些属性。如下：

    • name

      指令名（不包括v-前缀

    • value

      指令绑定的表达式计算后的值，如v-dir="1+1"，则value为2

    • oldValue

      指令绑定的前一个值，仅在update和componentUpdated钩子中可用。

    • expression

      字符串形式的指令表达式，即未经计算的表达式。如v-dir=1+1，则其值为"1+1"

    • arg

      传给指令的参数，是钩子函数的可选参数。如在指令v-dir:bar="1+1"中，则其值为bar。

    • modifiers

      指令的修饰符对象参数，包含指令的各修饰符的使用情况。如在指令v-dir.foo.bar="1+1"中，其值为{foo: true, bar: true}。

    需要注意的是，除了上面提到的这些属性中，除了el，其他都是只读的，不能进行修改。

    指令通常是绑定到一个JS表达式，但也可以绑定到一个对象字面量。形如v-dir="{label: true, hide: false}"。
