Vue系列（十七）：Vue Router基础

发表于 2018-08-22 | 分类于 前端 ， Vue |
字数统计 3,078 字 | 阅读时长 12 分钟

Vue Router 作为官方的路由器库，基本涵盖了一个前端SPA项目路由所需的所有功能。从路由的动态参数，嵌套路由，命名路由以及重定向等基础功能到懒加载，路由守卫等高阶功能，这一个库都可以说完全够用了。

作为一个Vue的官方指定库，风格肯定跟Vue 是兄弟俩，上手非常简单，只要理解了前端SPA路由的基本概念和逻辑，就可以愉快的使用起来。

跟Vue一样，简单容易上手的代价，就是当项目代码增长到一定规模，路由开始变得复杂起来时，它的各种路由子路由命名视图的嵌套，各种路由钩子，重定向，路径参数等等，再加上权限控制，路由动态跳转等等，混杂在一起，还是非常考验项目成员的维护和管理水平的。再加上没有类似Angular那样细致到模块的路由控制粒度，在多人开发情况下，还是比较容易造成混乱的。

当然，Vue本来的缺点，也不应该全怪罪到Vue Router身上。

从自身做起，对代码负责，提高自己对框架和库的了解程度以及技术水平，来尽量写出易维护的代码，才是应该去认真思考的问题，与大家共勉。

好了，不多说废话。这篇博客，就主要来介绍一下Vue-router的基本知识和使用。

嵌套路由

嵌套路由中，每一层路由对应一层嵌套的组件,可嵌套多层。
通过路由表中的children属性来定义路由的嵌套子层级，属性值为子层级的路由表

命名路由

某些路由的路径比较长时，我们在导航到其时会比较麻烦。可以通过为路径配置一个标志名称name来为路由命名。

重定向路由

通过配置redirect属性来实现访问某个路径时重定向到另一个路径。

关于重定向有一个需要注意的地方是路由的导航守卫只会作用在重定向的目标路径上，在被重定向的路径上不会触发。

路由别名

可以通过alias属性为路由起一个别名,从而实现多个URL路径访问一个路由的效果。

路径参数

在路由的路径中，我们经常会带上一些参数。例如user/22用于显示id为22的用户,user/33用于显示id为33的用户。通过动态路由参数，我们将一个组件映射到多个路由。

路径匹配规则

除了使用/:id来通配路径，还可以通过更高级的高级匹配模式来定义更复杂的路径到组件的映射关系。

具体可参考vue-router所使用的 path-to-regexp路径匹配引擎的介绍。
对于路径的模式匹配来说，如果URL可以匹配到多个路径，先定义的路径具有更高的优先级。

获取路径参数

通过当前页面的路由对象this.$route可以得到当前页面的路由路径参数。

除了params属性，路由对象$route还有以下属性:

  • path
    当前路由的路径字符串(不包含查询参数和锚点信息

  • query
    当前路由的查询参数对象
    若无查询参数，则为空对象

  • hash
    路由携带的锚点信息字符串

  • fullPath
    完整的包含查询参数和锚点信息的路径字符串

  • name
    当前路由的名称(若当前路由无名称则为undefined

  • redirectedForm
    当前路由的重定向来源路由名称

  • matched
    当前路由和它的嵌套的子路由的路由定义值数组（即我们传递给new VueRouter数组中与当前路由有关的部分组成的数组

另外需要注意的一点是，路由对象$route是只读的。

使用props将路由与组件解耦

通过URL传递参数，在组件中使用this.$route.params获取参数，意味着我们将路由和组件耦合在了一起。在其他路由的地方，我们就无法复用组件了。
可以通过路由定义中的props参数，来将路由参数和$route解除耦合。
使得我们在路由中携带的参数，作为组件的props的一个值传入，从而使我们在其他任何地方都可以使用该组件。

  • push

需要注意如果路由对象中有path属性，那么就意味着params已经被携带在了路径里，如果对象中还定义了params，则会被忽略。

在上面的HTML导航中我们提到的router-link组件的导航方式，导航时就是调用的push方法。因此它的to属性可以接受的参数格式和push方法完全一致。

push方法还可以接受两个可选的函数参数onComplete和onAbort, 这两个函数分别会在导航成功完成和未成功时(即导航到相同路由或另一个非目标路由）时调用。

  • replace
    replace与push的接受参数完全相同，效果也完全相同。唯一不同的时，push会在浏览器history记录中添加一条当前页面的记录，可以通过后退返回当前页。而replace则会直接使用导航到的目标页替换掉当前页的记录。

  • go
    接受一个数字作为参数，用于在历史记录中前进(参数为正)或回退数字参数(参数为负)步。

  • back
    在历史记录中后退一步

  • forward
    在历史记录中前进一步。
