Vue系列（九）：组件的Prop

发表于 2018-02-24 | 分类于 前端 ， Vue |

在前面的博客中介绍Vue组件时，我们提及到Vue 父子组件间的通信方式主要有两种：Prop 和 事件。

这篇博客中，主要介绍的就是前一种方式 —— 父组件通过props，向子组件传递数据。

Prop的使用

因为组件作用域的互相隔离，在子组件的模板中是无法直接引用父组件中的数据的。父组件需要通过 prop 才能将数据下发到子组件中。

验证

可以为组件的prop 指定验证规则，在传入的数据不符合要求时，Vue会发出警告。在开发给他人使用的组件时非常有用。

在为 prop 指定规则时，不能再使用字符串数组来定义props，而需要使用对象的形式。

  1. 当类型为null时，指允许任何类型
  2. type可以是 js 的原生构造器，也可以是自定义构造器函数，在检测类型时会调用 instanceof 检测。
  3. 检测发生在组件实例构造前，因此无法在default或validator函数中使用组件实例的其他属性。

单项数据流

父组件数据到 prop 的动态绑定是单向的。也就是说，父组件的数据变化会传递到子组件，但反向不会。

为了防止破坏这种单向数据流，造成数据状态的混乱和不一致，在子组件中不应该去改变prop的值，vue也会对这种行为做出警告。

子组件需要修改 prop 的情况主要有以下两种：

  1. 将 prop 当做初始值传入，作为子组件的局部数据来使用，不需要再响应 prop 的变化。

    这种情况下，可以定义一个组件的局部变量，并使用 prop 作为初始值初始化这个组件的局部变量，后续对局部变量的操作都与 prop 无关。

  2. 子组件需要对prop 原始值做一些处理后再使用

    这种情况下，可以定义一个计算属性，从而能够响应prop的变化并进行相应处理又不影响到父组件的原始值。

但需要注意的是，在以上两种情况下，如果 prop 为 js引用类型的数据，例如对象或数组，在子组件内改变它是会同时影响到父组件的状态的，此时可以考虑使用深拷贝等手段来保证单向的数据流。

非Prop 特性

为了给予组件接收数据更大的自由度，以应对组件被使用的各种场景。Vue的组件可以接收传入的任意特性，这些特性都会被添加到组件的根元素上。

添加data-scroll-bar-show="true"属性后，它会自动出现在组件的根元素上。

非prop特性的覆盖和合并

对于组件的大多数特性来说，传递给组件的特性值都会覆盖掉组件自身的设定值。

例外的是class 和 Style ，组件在对待这两个特性时，会采取合并(merge)的策略，将传入的特性值和组件自身的特性值进行合并操作。
