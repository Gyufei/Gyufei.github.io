v-model双向绑定

Vue 使用v-model指令在表单元素（<input> ,<textarea>等）上创建双向绑定。

在本质上，v-model指令只是一个语法糖，相当于对元素分别监听输入事件和更新数据。

同时需要注意以下几点：

  1. v-model 会根据控件类型自动选取正确的方法来更新元素。
  2. v-model 会忽略所有表单元素的 value，checked，selected 等特性的初始值，总是将 Vue 实例的数据作为数据来源。如果想设置初始值，可在Vue实例的data选项中声明数据的初始值。
  3. v-model 的事件触发不同于v-on:input的一点是不会在中文输入法组合文字的过程中更新数据。
  4. v-model 通常绑定到布尔值或静态字符串或静态字符串的数组。如果我们想将值绑定到Vue实例的动态属性上，可以使用v-bind绑定相应表单元素的value值 ,也可以为value绑定对象,数组等类型值 。

选择框

v-model不存在初始值时，<select> 元素会被渲染为未选中状态。这种情况会导致在 IOS 系统中无法触发change事件，用户无法选择第一个选项。

解决办法为：提供一个值为空的禁用选项。

修饰符

  • .lazy

    v-model默认在每次 input 事件触发时进行数据的更新和同步(除上述输入法组合文字时)。

    .lazy 修饰符 用于将v-model 转换为当触发 change 事件时同步。（即内容改变且失去焦点时同步)

  • .number

    .number用于自动将用户输入的数字值转换为数字类型（如果无法转换为数字类型则什么也不做）。

    需要注意的是，即使在设置type = number后，HTML 输入元素的值也总是返回字符串。

  • .trim

    .trim 用于自动过滤用户输入的首尾空白字符。
