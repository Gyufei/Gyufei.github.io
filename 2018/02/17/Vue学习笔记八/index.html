Vue系列（八）：组件的使用

发表于 2018-02-17 | 分类于 前端 ， Vue |
字数统计 1,498 字 | 阅读时长 6 分钟

组件作为Vue的灵魂功能，为前端开发提供了更大的空间和可能。

本篇博客主要介绍Vue组件的核心概念和使用方法。

组件简介

组件是 Vue.js 最为强大的功能之一。它相当于自定义的元素，主要作用为扩展HTML元素和封装可重用代码。组件也可以使用 is 特性来扩展原生的 HTML元素。

所有Vue组件同时也都是 Vue 实例，拥有和实例相同的生命周期钩子，可接受相同的选项对象（除了一些根级实例特有的选项）。

组件的注册

组件在注册之后即可作为自定义元素在实例的模板中使用。需要注意的一点是组件必须在初始化根实例之前注册。

组件有两种注册方式，即全局注册和局部注册。

全局注册

使用Vue.component(tagname,options) 来注册一个全局组件。

组件命名的推荐规则为 小写并包含一个短杠。

局部注册

组件可以局部注册在 Vue实例或其他组件的实例选项的 components中，也仅在其局部注册的作用域中可用。

组件的模板

Vue 中有多种方式来定义组件的模板。需要注意的一点是，在使用HTML模板时，在一些特殊的元素中使用组件可能会出现异常，例如<ul> , <ol> , <table> , <select> ,可能会出现渲染异常，需要使用 特殊的 is 特性来解决。

Vue定义组件模板的方式主要有以下几种：

  • 字符串模板

    即使用字符串格式的HTML，将其声明在组件的template属性中。

    此方法较难维护，唯一优点在于广泛的浏览器支持。

    另外也可以通过ES6 新增的模板字面量语法来定义多行模板。

    此方法更易阅读，但ES6可能存在兼容风险。

  • x-templates

    在类型为text/x-template 的script标签中定义模板。在定义组件时通过id来引用此类型模板。

    需要注意，x-template 模板定义需要在vue.js引入前。

  • 内联模板inline template

    通过为组件的 HTML 添加inline-template来向 vue 声明，其中内容为模板，而不是分发内容的slot。

    缺点同样是会将组件的定义分隔在两个地方或文件中。

  • 单文件组件

    单文件组件文件的文件扩展名为 .vue 。你可以将组件的 HTML，JS, CSS全部放置在此文件中，从而集中组件的所有关注点在相同的文件中。

    .vue 文件中包含三种类型的语言块。通过设置语言块的lang属性，可以使用非默认语言。通过配置工具的各种loader处理，可以将文件最终导出为一个vue组件对象。三种语言块的基本信息如下:

    • <template>

      默认语言为HTML。

      每个.vue文件最多包含一个<template>块。其中的内容将被提取并作为Vue组件的templates选项。

    • <script>

      默认语言为js。

      每个.vue 文件最多包含一个 <script> 块，块中的脚本必须导出组件对象。

    • <style>

      默认语言为css 。

      每个.vue文件中可以包含多个<Style>块。默认会将其中的样式动态加入编译生成文件的<head> 中，也可以通过配置将其提取到单个CSS文件中。

    缺点：需要使用构建工具（这真的能算缺点吗？^*^!)

  • render函数和 JSX 语法

    在一些特殊的情况下，可能需要使用渲染函数render来生成模板，为了简化render函数书写，也可以引入JSX语法。

组件的data

定义组件时可以使用大部分构造Vue实例时传入的各种选项。但需要注意的是：

组件的 data 必须是函数

这样在组件的复用时，才能基本保证各组件间的状态不因为引用问题而互相影响和冲突。但即使data为函数时，仍需要注意避免组件复用时共享到某些对象的引用。

组件的关系

在组件中使用组件，即组件间为父子关系，是组件使用中常见的情况。Vue设置了父子组件间的两种通信方式：

  • 父组件通过prop下发数据给子组件
  • 子组件通过事件传递信息给父组件

以上两种通信方式的具体实现和相关规则，因为篇幅原因，将放在后续两篇博客中进行详细介绍。
