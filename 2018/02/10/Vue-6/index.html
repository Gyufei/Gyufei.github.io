Vue系列（六）：事件处理

发表于 2018-02-10 | 分类于 前端 ， Vue |
字数统计 1,236 字 | 阅读时长 5 分钟

事件处理，作为网页开发中的重要组成部分，是所有前端框架都必须的一个实现。

在前端的事件处理中，如何做到关注点分离和代码的简洁优雅，是一个值得思考的问题。

本篇博客主要介绍了Vue框架事件处理的方法和注意事项。

监听和处理事件

Vue使用 v-on 指令监听 DOM 事件。

在事件触发时，执行相应JavaScript代码，有三种写法：

  1. 绑定到 JS 表达式

    在事件处理逻辑非常简单时，可以使用此种方法。在逻辑较为复杂时严重不推荐。

  2. 绑定到方法名称

    在事件处理逻辑比较复杂时，v-on 可以接收一个方法名称用于在事件触发时调用此方法。

    方法默认传入的参数 event 为 原生 DOM 事件

  3. 绑定到方法调用

    也可以在v-on 的内联js 语句中直接调用方法。

    在内联的方法调用时，如果需要访问原始的DOM事件，可以显式的传入特殊变量$event。

事件修饰符

为了使事件所绑定触发方法能更纯粹的处理数据逻辑，而不必再去处理常见的event.preventDefault() 或 event.stopPropagation() 等。vue提供了一系列的事件修饰符，使用方式形如v-on:click.stop。

事件修饰符有以下几种：

  • .stop 阻止事件的进一步传播
  • .prevent 阻止事件的默认行为（例如点击链接的跳转等）
  • .capture 使用事件捕获模式（即元素的事件在捕获阶段触发）
  • .self 只有当前元素自身才能触发事件(即只有在目标阶段触发)
  • .once vue 2.1.4 新增事件只触发一次。此修饰符特殊之处在于可以使用在自定义的组件事件上。
  • .passive vue 2.3.0 新增，用于在事件触发时默认行为立即发生，不必等待事件响应函数完成，可显著提高移动端性能。此修饰符的优先级高于.prevent。

事件修饰符可以串联使用。形如v-on:click.stop.prevent 。

按键修饰符

可以通过为v-on添加按键修饰符，来监听对应按键的键盘事件。例如v-on:keyup.13 ，即在keyCode为 13 的按键弹起时触发事件。

#自定义按键修饰符别名

可以通过Vue 的 Vue.config.keyCodes 方法来自定义按键修饰符别名。

Vue.config.keyCodes　为一个对象，对象中属性的属性名为按键修饰符别名，属性值为按键的keyCode

#预置的按键修饰符别名

Vue 为一些最常用的按键预置了别名，分别有：

  • .enter
  • .tab
  • .esc
  • .space
  • .delete
  • .up
  • .down
  • .left
  • .right

#自动匹配按键修饰符

在按键事件的原生DOM事件$event中，$event.key 即为按下的键名。

在 Vue 2.5.0 之后，可将此键名作为按键修饰符来使用（需转换为短横线形式），形如 <input @keyup.page-down="onPageDown"> ,在 $event.key === "pageDown" 时会被调用。

系统修饰符

系统修饰符主要用于系统按键的组合操作，例如alt + q，ctrl + 点击 等类似情况。

系统修饰符有以下这些：

  • .ctrl

  • .alt

  • shift

  • meta 在 Mac 上对应command键，在 windows 上对应win键

  • exact

    .exact用于更精确的控制系统修饰符组合的触发，当且仅当被其修饰的按键按下时才会触发。

#鼠标按键修饰符

有以下三个鼠标按键修饰符用于鼠标按键的事件触发：

  • .left
  • .right
  • .middle
