TypeScript(五):类

发表于 2017-05-13 | 分类于 前端 ， TypeScript |
字数统计 1,080 字 | 阅读时长 4 分钟

Ts 的类，大部分特性都是很多其他语言也有的相似范式特性，与Es6中新提出的类实现也非常类似。因此，如果接触过编程中类的一些基础概念的话，应该非常容易理解。

本篇博客就来主要介绍一下，Ts的类的一些基本知识。

类的基本实现

  • costructor　是类的构造函数
  • this 是类实例的引用
  • 通过new关键字来创建一个类的实例
  • 实例属性可以在构造函数中初始化，也可以在声明时直接赋值。

类的成员修饰符

与大部分具有类的语言一样，在Ts中，也存在public,protected,private三种修饰符用来指定类成员的可见性。

  • public, 表示成员对所有人都是可见的。类成员默认都是public的。
  • protected, 表示成员仅在当前类及其子类中可被访问。
  • privated, 表示成员仅在被定义的类中可被访问，不能通过类的实例或子类访问。

特别的，类属性也可以加上readonly修饰符，来将其限定为只读。
设置了只读的属性必须在属性声明或构造函数中被初始化。

修饰符和构造函数

为构造函数加上protected修饰符，则其只能在类的子类中被访问，也就是相当于将类限定为了只能被继承的基类。
为构造函数加上privated修饰符，则相当于将类限定为了即不能被继承，也不能被实例化的类（可以称其为没什么用类。

静态属性

可以通过static关键字，将属性声明为类的静态属性和方法。
类似于通过this来访问实例属性，可以通过类名来访问类的静态属性和方法。

类的继承

通过extends关键字来进行类的继承。并可以在子类中重写父类的方法。
typescript规定，子类如果定义了构造函数，必须在其中通过super来调用一次父类的构造函数。

抽象类

通过在类声明前加上abstract来将类声明为一个抽象类, 抽象类无法被实例化（但继承抽象类的子类可以）。
抽象类与接口类似，但相对于接口而言，抽象类可以包含成员的具体实现。

抽象方法

通过在抽象类的方法声明前加上abstract来将方法声明为一个抽象方法。
抽象方法与函数类型的接口类似，只需要定义方法的签名，不需要方法的具体实现。
在抽象方法前同样可以添加parvate等访问修饰。
在继承抽象类的子类中，必须实现抽象方法，否则就会报错。
