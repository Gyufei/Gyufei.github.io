TypeScript(三):接口

发表于 2017-05-03 | 分类于 前端 ， TypeScript |
字数统计 795 字 | 阅读时长 3 分钟

接口是Ts中的一个重要概念。

本篇博客主要介绍Ts中接口的基础知识。

接口定义

接口用于声明对象，函数或类的结构和类型。

  • 使用 interface 声明口
  • 使用属性名后的 ? 表示属性可选
  • 使用属性名前的 readonly 表示属性只读。

对象类型接口

对象字面量的特别之处

一般来说，定义了接口的函数只会检查传递的值是否符合了接口结构，不会再去检查它是否还包含额外的接口未定义的属性。

但是，当向定义了特定接口的函数传递对象字面量值时，则会对对象字面量做额外的多余属性检查，必须保证对象字面量的结构和接口完全一致，不能有额外的属性，否则就会报编译错误。

避免以上问题有三种方式:

  1. 将对象字面量赋值给一个变量再传递给函数
  2. 对对象字面量使用类型断言
  3. 使用接口的字符串索引签名来声明接口结构

索引

可以为接口增加可索引类型,包括一个索引签名和索引的返回类型。
索引签名有两种，字符串索引签名和数字索引签名，数字索引签名的返回值必须是字符串索引签名返回值的子类型（因为数字索引在js中等同于字符串索引。
同样的，索引类型也可以设置为只读来禁止对对象属性的赋值。

函数类型接口

接口除了声明对象的结构，也可以用来声明函数的结构。

类类型接口

接口也可以用来约束类的结构和类型，使用implements关键字。
接口中的属性声明用来约束类的实例属性，函数声明用来约束类的实例方法。
也就是说，接口只能用来约束类的实例属性和方法，无法约束类自身的构造函数，静态属性和静态方法。

接口继承

接口可以使用extends来继承其他任意个数的接口。通过继承，可以来创建多个接口的合成接口。

接口也可以继承类，在继承类时，接口相当于抽象出了类的结构，也包括类的private和protected成员。这两类成员只能被接口所继承类的本身或其子类实现。因此，当接口继承的类有私有成员时，接口也就只能被它所继承的类或其子类实现了。
