TypeScript(二):枚举

发表于 2017-04-18 | 分类于 前端 ， TypeScript |
字数统计 826 字 | 阅读时长 3 分钟

枚举也是Ts从静态类型语言中吸收的一个重要语言特性。作为一种基本的数据类型，枚举的应用场景还是比较多的，在我们需要一组固定的名称时，使用枚举即符合语义，又十分方便。

下面我对Ts中枚举的相关概念和知识做一个简单的介绍。

数字枚举

数字枚举可以为一组数字赋予友好的名称。
既可以通过数字值得到名称，也可以通过名称得到数字。
枚举默认从 0 开始自动增长计数。但也可以手动指定初始值或各枚举属性的具体值。

字符串枚举

可以使用字符串字面量作为枚举属性的值，字符串枚举是没有自增长行为的。

当然枚举也可以混合字符串和数字值，但并没有什么实用意义。

枚举成员值

枚举成员的值，可以是常量或者计算得出。
在以下情况时，枚举值被当做常量。

  • 没有值的枚举第一个成员，默认为 0
  • 其前一个成员的值为数字常量的，其值为前一成员值加 1
  • 其值为常数枚举表达式的，会在编译阶段求值，所以也是常量。有以下几种情况：
    • 成员值为字符串或数字字面量
    • 其值为之前定义的常量枚举成员的引用（可以在另外的枚举中
    • 其值为常数枚举表达式的四则运算和位运算。

枚举成员类型

当枚举成员的值均为字符串字面量或数字字面量时，枚举就拥有了特殊的类型语义。即我们可以声明某值类型只能是枚举成员的值。枚举的类型也即是所有枚举成员的联合类型。

反向映射

数字枚举成员具有反向映射的能力,可以通过数字值反向得到枚举成员名。

但需要注意，字符串枚举成员是没有反向映射的能力的。

const 枚举

const可以用来定义一个常量枚举，这种方式定义的枚举不同于常规枚举，它不会在编译后产生多余的枚举对象Js代码，而是直接将枚举值内联进编译后的代码。
const枚举的成员值只能为常量，不能包含计算成员。

外部枚举

使用declare声明的枚举为外部枚举，用来描述已存在的枚举类型的形状。对于外部枚举来说，比较特殊的一点是，在数字枚举中，当枚举成员无初始值时，则枚举成员就会被当做需要计算的成员，而不会在自动递增从而成为常数成员。
